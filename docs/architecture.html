BugLens Architecture

How the npm package + standalone API serves both Next.js and Vite projects

npm package (installed in every project)

npm package
buglens
npm install github:rilolabs/buglens
  • buglens/client — Provider, Widget, Overlay, Capture
  • buglens/server — Linear service, formatIssueBody
  • buglens/next — createBugLensHandler (optional)

two integration paths

Path A: Next.js projectse.g. SFF Grant Reporting
Next.js App
SFF Grant Reporting
Next.js 15 • App Router • Supabase
// app/layout.tsx
<BugLensProvider>
  {children}
</BugLensProvider>
// app/api/buglens/report/route.ts
export const POST = createBugLensHandler({
  authorize: async () => {
    // Supabase auth check
    return { authorized: true, email }
  }
})
  • Same-origin API route
  • Server-side email injection
  • No CORS needed
